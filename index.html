Web-site internal optimization for search results improving

@AndreiChaika
Internal optimization in SEO is a process that we follow in order to improve search results, and it could be done with adjusting site content. That basically helps search engines to improve site indexes.
  • Content-type part
  • Behaviour-type part (usability)
  • Technical part

Content-type art

  • Semanthic core develoment
  • Providing every page with specific key words
  • Content fulfilment
  • Pages coupling using hyperlinks

Behavior-type part (usability)

  • Content
  • Design
  • Abailability
  • Adaptation

Technical part

  • Critical
  • Major
  • Minor

Critical

  • Fulfil robots.txt and sitemap.xml files
  • Pages scanned
  • No broken pages and errors
  • No incorrect urls
  • No pages and content duplicates

Critical

  • Uinque title for each page with length from 70 up to 140 symbols
  • Uinque description for each page with length from 100 up to 160 symbols
  • At least one h1 block on each page, preferable length is from 3 up to 7 words
  • Correct redirects
  • Canonical URLs could be indexed and chain should not be broken

Critical

  • No broken images
  • All pages should be reachable
  • No more Flash
  • Correct Hreflang
  • Separate 404 page

Major

  • Response timeout should less than 500 ms
  • Content size should be greater than 500 symbols inside of «body» block (html tags are not counted).
  • Image tags should contain alt attribute
  • Image size should be less than 100 kBytes (mostly)

Major

  • No redirect chains are allowed
  • HTTPS protocol
  • Title and description should not be the samed
  • No analytic systems labels should be included in URLs

Minor

  • URL length should not be greater than 2k symbols. Ideally up to 100 symbols
  • Do not use unwanted symbols in URLs:
    (,_$@* and others
  • Hyphen symbols in URL should be used correctly. Bad example:

    https://site.com/category/-subcategory/testâ€”page-

Minor

  • URLs should be provided in lower case
  • No emodjis and special characters should be used neither in title, nor in description
  • HTML length should be less then 200 thousans symbols (html tags are included)
  • Content size should not be greater than 50 thousands symbols inside of «body» block (html tags are counted).

Minor

  • Text size should be greater than 10% of HTML size
  • Hyperlinks contain anchors
  • No cyclic links
  • Not more than 100 inner links per page
  • Not more than 10 external links per page
  • Microdata marks should present using JSON-LD, Microdata or RDF, using Schema.org
